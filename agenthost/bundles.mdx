# Bundles

AgentHost Bundles allow you to package multiple agents, services, and resources together to create comprehensive solutions for your customers. Whether you're building enterprise software suites, industry-specific solutions, or multi-agent workflows, bundles provide the flexibility and power you need.

## Bundle Configuration

### Bundle Structure

Define the architecture and relationships within your bundle:

```json Bundle Configuration
{
  "bundle_id": "enterprise_support_suite",
  "name": "Enterprise Customer Support Suite",
  "version": "2.1.0",
  "description": "Complete customer support solution with multi-agent workflow",
  
  "agents": [
    {
      "id": "intake_agent",
      "name": "Customer Intake Specialist",
      "type": "primary",
      "model": "gpt-4-turbo",
      "specialization": "customer_triage",
      "config": {
        "max_conversation_length": 20,
        "escalation_threshold": 3,
        "languages": ["en", "es", "fr"]
      }
    },
    {
      "id": "technical_agent",
      "name": "Technical Support Expert",
      "type": "specialist",
      "model": "claude-3-sonnet",
      "specialization": "technical_troubleshooting",
      "config": {
        "knowledge_base": "technical_docs_v2",
        "tools": ["screen_sharing", "remote_access"],
        "escalation_path": "human_engineer"
      }
    }
  ],
  
  "workflows": [
    {
      "name": "standard_support_flow",
      "trigger": "customer_message",
      "steps": [
        {
          "agent": "intake_agent",
          "condition": "always",
          "max_attempts": 3
        },
        {
          "agent": "technical_agent", 
          "condition": "technical_issue_detected",
          "handoff_criteria": ["issue_complexity > 7", "requires_tools"]
        }
      ]
    }
  ],
  
  "shared_resources": [
    {
      "type": "knowledge_base",
      "id": "company_knowledge",
      "accessible_by": ["intake_agent", "technical_agent"]
    },
    {
      "type": "customer_database",
      "id": "crm_integration",
      "permissions": "read_write"
    }
  ]
}
```

### Agent Coordination

Implement sophisticated coordination between agents:

<Tabs>
  <Tab title="Sequential Handoffs">
    **Use case**: Linear workflows where each agent handles a specific stage
    
    ```json Sequential Configuration
    {
      "coordination_type": "sequential",
      "handoff_rules": [
        {
          "from": "intake_agent",
          "to": "specialist_agent",
          "conditions": [
            "issue_category == 'technical'",
            "complexity_score > 5"
          ],
          "context_transfer": [
            "customer_info",
            "issue_description",
            "attempted_solutions"
          ]
        }
      ],
      "timeout_handling": {
        "max_wait_time": 300,
        "fallback_action": "escalate_to_human"
      }
    }
    ```
  </Tab>
  
  <Tab title="Parallel Processing">
    **Use case**: Multiple agents working simultaneously on different aspects
    
    ```json Parallel Configuration
    {
      "coordination_type": "parallel",
      "parallel_tasks": [
        {
          "agent": "research_agent",
          "task": "gather_background_info",
          "priority": "normal"
        },
        {
          "agent": "solution_agent",
          "task": "generate_recommendations", 
          "priority": "high",
          "depends_on": ["research_agent"]
        }
      ],
      "synchronization": {
        "wait_for_all": false,
        "timeout": 120,
        "partial_results": true
      }
    }
    ```
  </Tab>
  
  <Tab title="Collaborative">
    **Use case**: Agents working together in real-time
    
    ```json Collaborative Configuration
    {
      "coordination_type": "collaborative",
      "collaboration_model": "round_robin",
      "participants": [
        {
          "agent": "domain_expert",
          "role": "knowledge_provider"
        },
        {
          "agent": "communication_specialist",
          "role": "customer_interface"
        }
      ],
      "collaboration_rules": [
        {
          "when": "uncertainty > 0.7",
          "action": "consult_domain_expert"
        },
        {
          "when": "customer_frustrated",
          "action": "prioritize_communication_specialist"
        }
      ]
    }
    ```
  </Tab>
</Tabs>

### Container Orchestration

Deploy bundles using modern container technologies:

```yaml Docker Compose Example
version: '3.8'
services:
  bundle-orchestrator:
    image: agenthost/orchestrator:latest
    environment:
      - BUNDLE_ID=enterprise_support_suite
      - API_KEY=${AGENTHOST_API_KEY}
    ports:
      - "8080:8080"
    depends_on:
      - redis
      - postgres
      
  intake-agent:
    image: agenthost/agent:gpt-4-turbo
    environment:
      - AGENT_ID=intake_agent
      - SPECIALIZATION=customer_triage
    scale: 3
    
  technical-agent:
    image: agenthost/agent:claude-3-sonnet
    environment:
      - AGENT_ID=technical_agent
      - SPECIALIZATION=technical_troubleshooting
    scale: 2
    
  redis:
    image: redis:7-alpine
    
  postgres:
    image: postgres:15
    environment:
      - POSTGRES_DB=agenthost_bundle
```

### Kubernetes Deployment

Scale bundles with Kubernetes:

```yaml Kubernetes Manifest
apiVersion: apps/v1
kind: Deployment
metadata:
  name: support-bundle
spec:
  replicas: 5
  selector:
    matchLabels:
      app: support-bundle
  template:
    metadata:
      labels:
        app: support-bundle
    spec:
      containers:
      - name: bundle-orchestrator
        image: agenthost/orchestrator:v2.1.0
        ports:
        - containerPort: 8080
        env:
        - name: BUNDLE_CONFIG
          valueFrom:
            configMapKeyRef:
              name: bundle-config
              key: config.json
        resources:
          requests:
            memory: "512Mi"
            cpu: "250m"
          limits:
            memory: "1Gi"
            cpu: "500m"
---
apiVersion: v1
kind: Service
metadata:
  name: support-bundle-service
spec:
  selector:
    app: support-bundle
  ports:
  - port: 80
    targetPort: 8080
  type: LoadBalancer
```

## Bundle Management

### Version Control

Manage bundle versions and updates:

<AccordionGroup>
  <Accordion icon="code-branch" title="Version Strategy">
    **Semantic Versioning**: Use MAJOR.MINOR.PATCH format
    - **MAJOR**: Breaking changes in API or workflow
    - **MINOR**: New features, backward compatible
    - **PATCH**: Bug fixes and minor improvements
    
    **Example**:
    ```json
    {
      "version": "2.1.3",
      "changelog": {
        "2.1.3": ["Fixed escalation timeout bug", "Improved response accuracy"],
        "2.1.0": ["Added multi-language support", "New technical agent"],
        "2.0.0": ["Redesigned workflow engine", "Breaking API changes"]
      }
    }
    ```
  </Accordion>
  
  <Accordion icon="refresh" title="Update Management">
    **Rolling Updates**: Update agents one at a time
    **Blue-Green**: Maintain two environments for instant switching
    **Canary Releases**: Gradual rollout to subset of users
    
    ```json Update Configuration
    {
      "update_strategy": "rolling",
      "rollout_plan": {
        "stages": [
          {"percentage": 10, "duration": "1h"},
          {"percentage": 50, "duration": "4h"},
          {"percentage": 100, "duration": "ongoing"}
        ],
        "rollback_triggers": [
          "error_rate > 5%",
          "user_satisfaction < 4.0"
        ]
      }
    }
    ```
  </Accordion>
  
  <Accordion icon="history" title="Rollback Capabilities">
    **Automatic Rollback**: Based on performance metrics
    **Manual Rollback**: Admin-triggered immediate revert
    **Partial Rollback**: Rollback specific agents only
    
    ```json Rollback Policy
    {
      "auto_rollback": {
        "enabled": true,
        "triggers": [
          {"metric": "error_rate", "threshold": 0.05, "window": "5m"},
          {"metric": "response_time", "threshold": 5000, "window": "2m"}
        ]
      },
      "rollback_targets": [
        "previous_stable_version",
        "last_known_good_config"
      ]
    }
    ```
  </Accordion>
</AccordionGroup>

## Best Practices

### Design Principles

<Tip>
**Modular Design**: Create loosely coupled components that can be independently updated and scaled
</Tip>

<Tip>
**Clear Interfaces**: Define clear APIs and contracts between agents and services
</Tip>

<Warning>
**Avoid Tight Coupling**: Don't create dependencies that make it difficult to update individual components
</Warning>

### Performance Optimization

<AccordionGroup>
  <Accordion icon="rocket" title="Scalability">
    - Design for horizontal scaling
    - Implement proper caching strategies
    - Use asynchronous processing where possible
    - Optimize resource allocation
  </Accordion>
  
  <Accordion icon="shield" title="Reliability">
    - Implement circuit breakers
    - Use retry mechanisms with exponential backoff
    - Plan for graceful degradation
    - Monitor and alert on failures
  </Accordion>
  
  <Accordion icon="lock" title="Security">
    - Implement zero-trust architecture
    - Use encrypted communication between components
    - Regular security audits and updates
    - Proper access control and authentication
  </Accordion>
</AccordionGroup>

### Operational Excellence

<Note>
**Documentation**: Maintain comprehensive documentation for installation, configuration, and troubleshooting
</Note>

<Note>
**Testing**: Implement automated testing for all bundle components and workflows
</Note>

<Tip>
**Observability**: Implement comprehensive logging, metrics, and tracing across all components
</Tip>
